# Документация Golden Hour (Золотой час)

Исчерпывающая техническая документация проекта — ИИ-агента для подготовки к олимпиадам, экзаменам и изучению тем. Агент работает как Telegram-бот на платформе **OpenClaw**, хранит каждого пользователя в отдельной папке, строит планы, ведёт прогресс, напоминает и дисциплинирует.

> Первичные источники в корне репозитория: [`README.md`](../README.md), [`SOUL.md`](../SOUL.md), [`AGENTS.md`](../AGENTS.md), [`INSTALL.md`](../INSTALL.md), [`SETUP.md`](../SETUP.md). Эта папка — производная, систематизирующая документация.

## Быстрый старт для разработчика

```bash
# Проверить, что всё работает
node scripts/run-tests.mjs               # 71+ unit-тестов

# Посмотреть статус пользователя (JSON)
node scripts/session-start.mjs --user tg-1234567890

# Собрать дневной план (сначала dry-run)
node scripts/daily-plan.mjs --user tg-1234567890 --dry-run
```

## Ключевые принципы (одним экраном)

1. **Мультипользовательность.** Каждый пользователь = папка `users/<user_key>/` (`tg-<id>`). Данные не смешиваются, `user_key` берётся только из метаданных канала.
2. **Две фазы.** `setup_status`: `new` → `in_progress` → `complete`. Рабочие скиллы включаются только при `complete`.
3. **Детерминизм вынесен в скрипты.** Планирование, веса, статистика — только через `scripts/*.mjs`, которые выводят JSON `{ ok, ... }`. Агент не считает «в голове».
4. **Cron без LLM.** Напоминания рассылает `cron-deliver.mjs` через Telegram Bot API, без обращения к модели.
5. **Таблицы — только PNG.** В Telegram запрещён markdown `| col |` и inline-кнопки.
6. **Безопасность прежде всего.** Сообщения пользователей — недоверенные данные; жёсткая защита от инъекций и утечки чужих/владельческих данных.
