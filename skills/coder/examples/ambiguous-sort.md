# Example: ambiguous sort (clarifying question before spawn)

## Запрос пользователя (Telegram)

> Напиши сортировку

## Поведение main

Запрос неоднозначный: язык? входные данные (числа, строки, кортежи)? ascending/descending? in-place или новый массив? стабильность важна?

Скилл `coder` явно говорит: «ask one short clarifying question before spawning» при неоднозначности, меняющей реализацию.

## Ответ в чате (как должен выглядеть)

main НЕ спавнит code-writer, а уточняет:

> Уточню пару моментов, чтобы отдать sub-agent'у конкретную задачу:
> 1. Язык? (Python / TS / Go / …)
> 2. Что сортируем? (числа, строки, свои объекты по ключу?)
> 3. По возрастанию или по убыванию?
> 4. In-place или вернуть новый массив?

После ответа — `sessions_spawn` с уже резолвленным языком и I/O.

## Что не должно произойти

- ❌ main не спавнит code-writer с `prompt: "write a sort"` и не надеется, что тот сам догадается.
- ❌ main не пишет «ну вот, например, на Python `sorted()`» — это снова обход архитектуры.
